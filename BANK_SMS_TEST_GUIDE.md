## 🔍 故障排查

### 问题1：短信未被捕获

**检查项：**
1. 通知使用权是否已开启？
   ```
   设置 → 通知与控制中心 → 通知使用权 → 小小记账本通知监听
   ```

2. 短信应用包名是否被监听？
   - 小米设备：`com.android.mms`、`com.miui.sms`、`com.xiaomi.sms`

3. 银行短信捕获是否已开启？
   - TinyLedger → 设置 → 自动记账 → 银行短信捕获

**解决方法：**
- 关闭并重新开启通知使用权
- 重启 TinyLedger 应用
- 查看日志确认服务状态

---

### 问题2：短信被捕获但未解析

**检查日志：**
```bash
adb logcat | Select-String -Pattern "SmsTransactionParser"
```

**可能原因：**
1. 短信格式不符合已知模板
2. 缺少关键关键词（如"建设银行"、"支出"、"元"）
3. 金额格式异常

**解决方法：**
- 确认短信包含银行标识（如【建设银行】）
- 确认包含交易类型词（支出/收入/消费/到账）
- 确认金额格式正确（如"100.00元"）

---

### 问题3：解析成功但未弹出通知

**检查项：**
1. 是否开启了无感模式？
   - 无感模式：直接记账，不弹通知
   - 待确认模式：弹出确认通知

2. 通知权限是否已授予？
   ```
   设置 → 应用管理 → TinyLedger → 权限管理 → 通知
   ```

**查看当前模式：**
```bash
adb shell "run-as com.tinyledger.app cat /data/data/com.tinyledger.app/shared_prefs/notification_settings.xml"
```

查找 `seamless_auto_accounting_enabled`：
- `true` = 无感模式
- `false` 或不存在 = 待确认模式

---

### 问题4：重复记账

**正常情况下不会重复，因为：**
1. 通知监听服务优先处理
2. 短信接收器检测到通知监听活跃时会跳过

**日志确认：**
```
SmsReceiver: Notification listener active for bank SMS, skipping SmsReceiver auto-accounting to avoid duplicates
```

**如果仍然重复：**
- 检查是否有多个监听服务
- 重启应用
- 查看日志中的去重逻辑

## 💡 提示和技巧

### 1. 快速查看最近日志
```bash
adb logcat -d | Select-String -Pattern "TxNotifService" | Select-Object -Last 20
```

### 2. 导出完整日志
```bash
adb logcat -d > bank_sms_test_%date:~0,4%%date:~5,2%%date:~8,2%.log
```

### 3. 查看短信数据库
```bash
adb shell "run-as com.tinyledger.app ls /data/data/com.tinyledger.app/databases/"
```

### 4. 重启通知监听服务
```bash
adb shell am force-stop com.tinyledger.app
adb shell am start -n com.tinyledger.app/.MainActivity
```
